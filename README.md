# WildWooHooShop

The repository for **shop.wildwoohoo.com**, the online shop for WildWooHoo Studio.

Built as a Jekyll site, deployed via GitHub Pages. GitHub Pages auto-builds on push, no GitHub Action required.

- **Main site:** [wildwoohoo.com](https://wildwoohoo.com)
- **Shop domain:** shop.wildwoohoo.com (this repo)
- **Owner:** Dr WELI / WildWooHoo Studio (ABN 61 239 662 726)

## Architecture in one paragraph

Layouts in `_layouts/`, partials in `_includes/`, brand and nav data in `_data/`, categories as a Jekyll collection in `_categories/`, products (when they land) as a Jekyll collection in `_products/`. Update one file, the change propagates to every page. CSS is split between brand tokens (`css/wwh-brand-tokens.css`, a verbatim mirror of the main site) and shop-local styles (`css/shop.css`).

## How to make common changes

### Add a new category

Create `_categories/[slug].html` with front matter and content body. Reference any existing file (e.g., `_categories/wear.html`) for the schema. Jekyll generates the page at `/categories/[slug]/` and the landing-page card grid auto-includes it.

### Add a new product (when products exist)

1. Copy `_template/product-page.html` content into `_products/[product-slug].html`.
2. Replace every `REPLACE-ME` placeholder in the front matter and body.
3. Drop 7 product images into `/assets/img/[product-slug]/` (`01-hero.jpg` through `07-model.jpg` — see `_template/README.md` for the order rationale, per shop immersion Insight 6).
4. The page builds at `/products/[product-slug]/` automatically.

### Change the nav

Edit `_data/nav.yml`. Every page's header and footer nav updates.

### Change the brand metadata (email, AoC, social, etc.)

Edit `_data/site.yml`. Every page that references that field updates.

### Change the header or footer markup

Edit `_includes/header.html` or `_includes/footer.html`. Every page updates.

## Local development

Requires Ruby 3.1+ (the system Ruby on older macOS may be too old).

```bash
cd ~/Studio/Websites/WildWooHooShop
bundle install
bundle exec jekyll serve
# Site runs at http://localhost:4000
```

If you don't have a modern Ruby installed locally, that's fine — push to GitHub and Pages will build it. Build status: https://github.com/Dr-WELI/shop-wildwoohoo-com/actions or the deploy tab in the repo.

## Recommended phased launch (see STRATEGY-SUMMARY.md)

This scaffold is intentionally backend-agnostic. The recommended path:

- **Phase 1 (Q3 2026):** open with digital downloads, use **Lemon Squeezy** embedded buy buttons on this static scaffold. AU$0/mo fixed cost. Lemon Squeezy is the merchant-of-record so AU GST + EU VAT + US sales tax are handled for you.
- **Phase 2 (Q4 2026):** add POD merch via **Shopify Lite** (AU$9/mo) + Printful Australia. Embed Shopify Buy Buttons on this scaffold.
- **Phase 3 (2027, only when justified):** migrate to **Shopify Basic** (AU$45/mo) when SKUs > 15, or limited drops live, or revenue > AU$3K/mo.

For books specifically, **Lulu Direct** (Shopify-native book POD) lets you sell hardcover and paperback from this storefront, printed and shipped on demand, keeping your imprint. See the books-pathway notes in your conversation history.

## Discipline this repo inherits

From `~/Studio/CLAUDE.md` and `~/Studio/Websites/CLAUDE.md`:

- **Brand-domain rule.** WildWooHoo content lives on `wildwoohoo.com` and `shop.wildwoohoo.com`; Dr WELI content lives on `drweli.com`. Don't cross the streams.
- **AU English.** behaviour, programme, centre, organise, realise, colour.
- **Anti-AI-tell.** No em-dashes. No curly quotes. No Furthermore / Moreover / Additionally / In conclusion / Importantly / leverage / robust / delve / tapestry.
- **No overclaim.** No "show-stopping", "vanguard", "next big thing".
- **No re-introducing struck terms.** "Brain·Body·Planet" stays struck. "Living Wild" stays struck as a framing word.
- **Existing copy is good.** Lean on what the strategy docs already say. Don't rewrite.
- **No live-site changes** without WELI's explicit go.
