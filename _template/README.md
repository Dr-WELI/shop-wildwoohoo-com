# Product page templates

This folder holds the reference templates for product pages. It is excluded
from the build (`_template/` is in the `_config.yml` exclude list), so nothing
in here ever publishes.

- `product-page.html` - generic products: digital downloads, wear, prints, tools.
- `book-product.html` - the children's-books line: booklets and slipcase sets.

## Steps to add a new product

1. **Copy the template** to `_products/[product-slug].html`. Products are a
   Jekyll collection; the page auto-builds at `/products/[product-slug]/`.
   Do NOT create folders under `/categories/` - that was the pre-collection
   scheme and no longer exists.
2. **Replace every `REPLACE-ME`** in the front matter and body. Search for
   stray `REPLACE-ME` before you finish.
3. **Set the `category` key** to one of the five category slugs:
   `childrens-books`, `educational`, `wear`, `prints`, `tools`. This is what
   makes the product appear on its category page - the category layout builds
   its grid from `site.products | where: 'category', page.slug`, sorted by
   `order`. (There is no `merch` category; it split into `wear` and `prints`.)
4. **Drop product images** into `/assets/img/[product-slug]/` per the 7-image
   convention below.
5. **Never hand-edit `sitemap.xml`.** It is generated by the `jekyll-sitemap`
   plugin at build time and is not a file in this repo.
6. **Editorial chain.** Full chain (writer -> reviewer -> chief-editor) before
   the page goes live. See `~/Studio/CLAUDE.md`.

## Front-matter contract

Field names use underscores (`made_in`, `ships_from`, `price_compare_at`) and
AU spelling: the key is `fulfilment`. An older version of this template said
`fulfillment` with hyphenated field names; the live includes read the
underscore + AU-spelling keys only.

### Required on every product

| Field | Values | Notes |
|---|---|---|
| `title` | text | The product name as the customer searches |
| `lede` | text | 1-2 sentence hero opening |
| `description` | text | 150-160 char hand-written meta description |
| `price` | AUD integer | Round figures per Insight 8 |
| `category` | `childrens-books` / `educational` / `wear` / `prints` / `tools` | Must match a category slug - this is what makes the product appear on its category page |
| `order` | integer | Sort position in the category grid |
| `fulfilment` | `digital` / `pod` / `inventory` | Picks the tax line and the delivery disclosure line in the buy panel |
| `availability` | `live` / `in_stock` / `preorder` / `out_of_stock` | Drives the buy panel; a missing key renders the preorder state |

### The buy-state contract

The buy panel (`_includes/buy-state.html`, included by both product layouts)
crosses `availability` with `checkout_url`:

- `availability: live` plus `checkout_url` renders a real Buy now link.
  The anchor works as a plain Lemon Squeezy hosted-checkout link; lemon.js
  (loaded by the default layout only when `checkout_url` is set) enhances it
  to an overlay.
- `availability: live` without `checkout_url` renders "Available now" with a
  disabled "Cart opening soon" button.
- `in_stock`, `preorder` and `out_of_stock` render honest state lines with
  register-interest mailtos. No scarcity counters, no urgency timers, ever.

### Optional keys

| Field | Values | Notes |
|---|---|---|
| `checkout_url` | URL | Lemon Squeezy hosted-checkout URL; presence plus `availability: live` renders the real buy button |
| `related` | list of `{title, url, reason}` | Manual cross-sell, max 3, per Insight 35. Rendered by `related-products.html`; no algorithmic resolution |
| `age_band` | text, e.g. `7-8` | Children's products only; switches on the safety note (per Insight 23) |
| `safety` | list from `supervision`, `wash`, `small_parts` | Opt-in lines on the safety note; the copy lives in `_data/policies.yml` |
| `price_compare_at` | AUD | Only for documented discounts per Insight 21 |
| `breadcrumb` | text | Hero stamp, e.g. `Children's books - Kangaroo Time`; omit and no stamp renders |
| `og_image` | path | Falls back site-wide to `/assets/img/og-shop.png` |

### Spec-grid fields (product-meta.html)

The product layout renders a spec grid from whichever of these exist:
`format`, `files`, `size`, `licence`, `material`, `fit`, `made_in`,
`ships_from`, `care`. Set them in front matter; never hand-write spec
paragraphs in body copy.

Books use the richer book fields modelled in `book-product.html` (`isbn`,
`imprint`, `pages`, `languages`, `companion_code`, `combo_contents`,
`icip_status`). `isbn: TBC` is the internal placeholder; the spec grid hides
it until a real ISBN is allocated.

## The 7-image convention (per Insight 6)

Seven images per product, shot in this order for the photography session,
into `/assets/img/[product-slug]/`:

| Slot | File | What it shows |
|---|---|---|
| 1 | `01-hero.jpg` | Hero shot, product centred |
| 2 | `02-scale.jpg` | Scale, in hand or on body |
| 3 | `03-detail.jpg` | Close-up of the distinctive feature |
| 4 | `04-context.jpg` | Lifestyle, brand-specific context |
| 5 | `05-alternate.jpg` | Back or alternate angle |
| 6 | `06-packaging.jpg` | Packaging as product |
| 7 | `07-model.jpg` | Model or scale shot (optional) |

No stock photography (per Insight 33). Every image is commissioned or shot by
the studio.

## Title convention (per Insight 4)

`WildWooHoo | [Product Name] | [Variant]`

Max 70 chars for the Google product card. Aim 55-65 incl separators.

## Meta description (per Insight 5)

150-160 chars exactly. Hand-written. Never auto-generated by Shopify or any
other tool. Pattern: `[product type]` + `[one specificity competitor lacks]` +
`[soft commitment: price, ships-from, or use-context]`.

## Copy length (per Insight 34)

- Tees / posters / stickers / totes: 80-120 words body copy
- Zines / CDs / vinyl / books: 200-300 words body copy

## Pricing (per Insight 8)

Round AU$ figures. AU$45 not AU$44.99. Never hard-code price in body copy
(per Insight 27): price renders only in the buy panel, nowhere else.

## Sustainability claims (per Insights 2, 3, 7)

Name the supplier / spec / certification inline. No vague claims. Acknowledge
at least one limit inline if positioning as eco/sustainable.

## What the layouts append automatically

Do not duplicate any of these in body copy:

- the buy panel: price, tax line, availability state, delivery disclosure
- the spec grid (`product-meta.html` on product pages, `book-meta.html` on books)
- related products, from the `related` key
- the children's safety note, from `age_band` + `safety`
- the policy strip, which carries the AU Consumer Law sentence on every
  product and book page

## What the templates do NOT do

- No cart. Checkout is per-product hosted checkout (Lemon Squeezy) while the
  shop is static; Shopify Buy Buttons arrive in Phase 2.
- No multi-currency. Prices are AUD only until Shopify Markets (Phase 3).
- No analytics pixels. GA4, Meta Pixel, TikTok Pixel and Plausible get added
  at backend-migration time, not now.
